<p><a href="https://promptql.hasura.io/">PromptQL</a> is a data access agent that creates and executes query plans to retrieve data with 100% accuracy for business critical applications. By integrating web search in PromptQL, you can enrich your private data – stored in any systems like Snowflake, BigQuery, PostgreSQL, MongoDB, or accessed through APIs – with current real-time details from the web.</p><p>For example, you might query customer records in PromptQL and supplement that data with the latest public updates about those companies or individuals. Access to web search just opens up a lot of use cases.</p><blockquote>New to PromptQL? Please check out our <a href="https://promptql.hasura.io/">website</a> and documentation for a quick <a href="https://promptql.hasura.io/docs">Getting Started Guide</a>.</blockquote><figure class="kg-card kg-image-card kg-card-hascaption"><img alt="Enrich private data with Web Connectors" class="kg-image" height="1296" src="https://hasura.io/blog/content/images/2025/03/Web-connectors.png" width="1410" /><figcaption>Enrich private data with Web Connectors</figcaption></figure><p>In this post, I’ll explain some learnings from implementing web search connectors, showcasing various options available to you, and providing simple TypeScript examples to help you integrate these tools into your PromptQL setup. I also have some personal recommendations and notes on which tools work for what use case.</p><h2 id="web-search-tooling-in-the-context-of-genai">Web search tooling in the context of GenAI</h2><p>As I worked on integrating web search into PromptQL, I explored various tools for real-time search and AI-powered retrieval. Some had APIs and some didn’t. But before I delve deep, I wanted to point out that there are two distinct types of search tools.</p><ul><li>First, <em><strong>traditional web search engines</strong></em> like Brave Search and DuckDuckGo work by executing a keyword-based query and returning a set list of results – typically around 10 links for further exploration. You can feed the LLM these 10 results for some summary based on title/description or some URL extraction tasks.</li><li>In contrast, AI-powered search retrieval systems, such as Perplexity and Google Gemini Grounded with Google Search, <em><strong>integrate a language model with the web search functionality</strong></em>. They do not return a dump of search results necessarily but analyze and summarize the information, delivering a concise and real-time response.</li></ul><h2 id="web-search-tooling-landscape">Web search tooling landscape</h2><p>Alright, so with the above distinction out of the way, here are my observations about these tools.</p><p>I wired up some TypeScript code for the following search tools:</p><blockquote>A quick reminder: You can write custom TypeScript or Python functions – using Hasura DDN Connectors – to implement any business logic you need, and then expose those functions directly to PromptQL.</blockquote><h3 id="brave-search">Brave Search</h3><p>Brave Search is a solid option if you need classic search results. Its API returns detailed information such as titles, descriptions, URLs, timestamps, and other payloads. <a href="https://api-dashboard.search.brave.com/app/documentation/web-search/responses#Result">Refer here for their documentation</a>.</p><h3 id="duckduckgo-search">DuckDuckGo Search</h3><p>An alternative to Brave Search, DuckDuckGo Search supports web search in real time to query and return a list of search results that PromptQL can operate on. They have an API and client SDKs that you can use to get search results.</p><h3 id="google%E2%80%99s-gemini-api-grounding-with-search">Google’s Gemini API + Grounding with search</h3><p>This combination leverages Google’s massive search index and the performance improvements of Gemini 2.0 Flash models. By grounding the output of Gemini Flash models with the traditional Google Search tool, it ensures you receive fast, high-quality, and up-to-date information.</p><h3 id="perplexity-ai">Perplexity AI</h3><p>Perplexity delivers impressive results on its user interface; however, its API is currently based on older / inferior models. This limitation can make it slightly less reliable for applications that depend entirely on real-time, up-to-date data. There is probably an ad monetization play here to limit what you can do with the API.</p><h3 id="seachgpt-by-openai-and-other-tools">SeachGPT by OpenAI and other tools</h3><p>I wanted to try out SearchGPT, but unfortunately, it didn’t have an API and I believe it is a wrapper over Bing Search with AI-powered responses. Similar to Perplexity UI, the responses are really good here with their UI, but it’s bummer they don’t have an API.</p><p>I briefly explored very specific search tools like Exa Web (which is closer to real-time search but isn’t always accurate) and Tavily (which wasn’t reliable in people search) and didn’t find them better or more accurate than the above options.</p><h2 id="web-search-connector-quickstart-guides">Web search connector quickstart guides</h2><p>In this section, I’ll link to the implementations of these web search functions, most likely in TypeScript or Python with their respective source code.</p><!--kg-card-begin: html--><table style="border: medium;
